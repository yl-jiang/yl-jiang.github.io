函数定义

函数的定义必须包含4部分：

  1. 返回类型
  2. 函数名
  3. 参数列表
    • 参数类型
    • 参数名称
  4. 函数体

函数声明

函数声明必须包含：

  • 函数返回类型
  • 函数参数类型
  • 参数名可写可不写

注意事项

  • 函数必须先被声明，然后才能被调用。函数的声明让编译器得以检查后续出现的使用方式是否正确——是否有足够的参数、参数类型是否正确，等等。函数声明不必提供函数体，但必须指明返回类型、函数名、以及参数列表。

  • 非void函数必须有return语句。

  • 函数内定义的对象，只存在于函数执行期间(static除外)，函数执行完便会被丢弃，local object也不复存在，如果将这些对象的地址返回，会导致运行出现错误。

终止程序(exit)

必须传给 exit() 一个值，此值将成为程序结束时的状态值。

#include<cstdlib>

// ...
exit(-1)
// ...

查询内置数值类型的表示范围

#include<limits>

int max_int = numeric_limits<int>::max();
double min_dbl = numeric_limits<double>::min();

调用函数

两种参数传递方式：

  1. 传址(by reference) —— 类似于Python中的”in-place”
  2. 传值(by value)

当我们调用一个函数时，会在内存中建立一块特殊的区域，称为“program stack”。这块特殊区域提供了每个函数参数的存储空间。它也提供了函数所定义的每个对象的内存空间——我们将这些对象称为local object。一旦函数完成，这块内存就会被释放掉，或者说是从程序堆栈中被pop出来。

当我们以by reference方式将对象作为函数参数传入时，对象本身并不会复制出另一份——复制的是对象的地址。函数中对该对象进行的任何操作，都相当于是对传入的对象进行间接的操作。将参数声明为reference的理由之一是，希望得以直接对所传入的对象进行修改；将参数声明为reference的第二个理由是，降低复制大型对象的额外负担。

当我们以传值的方式调用函数，传递给函数的参数会被复制一份，原参数对象和副本参数之间没有任何关联。

#include<iostream>
#include<vector>
#include<cstddef>
#include<string>
using namespace std;

void display(const vector<int> &);  // 打印函数；直接传入打印对象的地址，速度会更快；使不使用const都不会对程序的运行造成影响。这里加上const，是为了向阅读此程序的人说明，虽然这个函数传入的是对象的reference，但在这个函数里面我们并不打算对其进行修改。
void swap(int &, int &);  // 交换操作
void bubble_sort(vector<int> &);  // 冒泡排序

void display(const vector<int> &vec){
    for (size_t ix = 0; ix < vec.size(); ++ix)
        cout << vec[ix] << " "; 
}


void swap(int &val_1, int &val_2){
    int temp = val_1;
    val_1 = val_2;
    val_2 = temp;
}

void bubble_sort(vector<int> &vec){
    for (size_t ix = 0; ix < vec.size(); ++ix){
        for (size_t jx = ix + 1; jx < vec.size(); ++jx){
            if (vec[ix] > vec[jx]){
                swap(vec[ix], vec[jx]);
            }
        }
    }
}

int main(){
    int arr[8] = {8, 34, 3, 13, 1, 21, 5, 2};
    vector<int> vec(arr, arr + 8);
    cout << "vector before sort: ";
    display(vec);
    cout << "\n";
    bubble_sort(vec);
    cout << "after sort: ";
    display(vec);
    cout << "\n";

    getchar();
}

上面的程序为冒泡排序算法，其中用户创建的vector必须以“by reference”的方式传递给 bubble_sort() 函数，在 bubble_sort() 内又必须以“by reference”的参数传递方式将参数传递给函数 swap()。

默认参数

  • 默认值的解析(resolve)操作由最右边开始进行。如果我们为某个参数提供了默认值，那么这一参数右侧的所有参数都必须也具有默认参数值才行。
// 错误的默认参数值书写
void display(ostream &os = count, const vector<int> &vec){
    // ...
}
  • 默认值只能够指定一次，可以在函数声明处，也可以在函数定义处，但不能够在两个地方都指定。

作用域及范围

  • 对象在程序内的存活区域称为该对象的scope
  • 对象如果在函数以外声明，具有所谓的file scope。具有file scope的对象，从其声明点至文件末尾都是可见的。file scope内的对象也具备所谓的static extent，意即该对象的内存在main()开始执行之前便已经分配好了，可以一直存在到程序结束。
  • 在函数内部声明的的变量，为局部非静态对象，其在每次调用函数时建立，并在函数结束时被丢弃。
const vector<int> *fibon_seq(int size){
    vector<int> elems; // 局部非静态对象

    // ... (对elems里的元素进行增/删/改等操作)

    return &elems; // 这是错误的，因为elems所在的内存会在函数执行完毕后被销毁，因此自然也就无法得到其地址
}
  • 局部静态对象在其定义出生成，其所处的内存空间，即使在不同的函数调用过程中，也依然持续存在。
const vector<int> *fibon_seq(int size){
    static vector<int> elems; // 局部静态变量

    // ... (对elems里的元素进行增/删/改等操作)

    return &elems; // 正确，elems为局部静态对象，其内存在整个main()函数执行结束之前都会一直存在
}

inline函数

声明为inline的函数，在编译时会将其编译到其被调用的函数内，从而达到形式上的分离而实质上的统一。

// 定义inline函数
inline void display(string input_type, string msg){

    // ...
}

// 声明inline函数
inline void display(string, string);

重载函数

参数列表不同(可以是参数类型不同，也可以是参数数量不同)的两个或多个函数，可以拥有相同的函数名称。

#include<iostream>
#include<string>
#include<vector>

using namespace std;

int max(int& val_1, int& val_2){
    int max_val = (val_1 > val_2) ? val_1 : val_2;
    return max_val;
}

double max(double& val_1, double& val_2){
    double max_val = (val_1 > val_2) ? val_1 : val_2;
        return max_val;
}

string max(string& str_1, string& str_2){
    string max_str = (str_1 > str_2) ? str_1 : str_2;
    return max_str;
}

int main(){
    int val_1 = 10;
    int val_2 = 5;
    cout << "two int: " << max(val_1, val_2) << endl;

    string str_1 = "abc";
    string str_2 = "abd";
    cout << "two string: " << max(str_1, str_2) << endl;

    double flt_1 = 2.5;
    double flt_2 = 3.6;
    cout << "two double: " << max(flt_1, flt_2) << endl;

    getchar();
}

函数模板

程序代码的主体不改变，仅仅改变其中用到的数据类型。

#include<iostream>
#include<string>
#include<vector>

using namespace std;
template<typename elemType>

elemType max(elemType& val_1, elemType& val_2){
    if (val_1 < val_2){
        return val_2;
    }
    else{
        return val_1;
    }
}


int main(){
    int val_1 = 10;
    int val_2 = 5;
    cout << "two int: " << max(val_1, val_2) << endl;

    string str_1 = "abc";
    string str_2 = "abd";
    cout << "two string: " << max(str_1, str_2) << endl;

    double flt_1 = 2.5;
    double flt_2 = 3.6;
    cout << "two double: " << max(flt_1, flt_2) << endl;

    getchar();
}

个人感觉template<typename elemType>有点类似于Python中的格式化字符串的意思，只不过它将我们定义的elemType替换为某种数据类型，该数据类型根据传入的参数中第一个遇到elemType的参数决定。

例如当执行max(val_1, val_2)，由于函数max()的第一个位置参数就为elemType所定义，而我们传入的val_1的数据类型为int，因此在本次调用中编译器就会将elemType当作int，如果我们传入的val_2不是int型的数据，那么就不能通过编译。

函数指针

const vector<int>* (*fun_ptr)(int); // 指针'fun_ptr'指向返回值为(某个类型)指针的函数

函数指针fun_ptr指向一个返回类型为const vector<int>*(表示一种指针)，并只含一个int类型的参数的函数。
